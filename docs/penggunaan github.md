# Penggunaan Github dalam Software Quality

## Deskripsi
Github digunakan sebagai alat bantu dalam proses pengembangan dan pengujian software untuk menyimpan source code,dokumentasi, serta mempermudah pengelolaan project secara terstruktur.

---

## Fungsi Github

### 1. Repository
Repository digunakan untuk menyimpan seluruh file project,baik source code maupun dokumentasi.

Pada project ini, repository berisi:
- Source code sistem CBT (folder src)
- Dokumen SKPL (folder skpl)
- Hasil review software (folder review)
- Dokumentasi pengujian (folder testing)

---

### 2. Version Control (Commit)
Setiap perubahan pada project disimpan dalam bentuk commit, sehingga semua aktivitas dapat dilacak.

Contoh commit:
- add: struktur folder project
- add: source code CBT
- add: dokumentasi SKPL
- add: testing documentation

Dengan adanya commit, perubahan dapat dikontrol dan jika terjadi kesalahan dapat dikembalikan ke versi sebelumnya.

---

### 3. Dokumentasi
Github digunakan untuk menyimpan berbagai dokumentasi penting, seperti:
- SKPL (Spesifikasi Kebutuhan Perangkat Lunak)
- Software Review (Walktrough, Inspection, Technical Review)
- Test case dan hasil pengujian

Dokumentasi ini membantu dalam memahami sistem serta memudahkan proses evaluasi software.

---

### 4. Pengujian Software
Hasil pengujian disimpan pada folder testing,meliputi:
- Black Box Testing
- White Box Testing
- Grey Box Testing
- Test Case 
- Test Result

Dengan penyimpanan ini, hasil pengujian dapat dengan mudah dilacak, dianalisis, dan dikembangkan.

---

### 5. Kolaborasi
Github memungkinkan beberapa anggota tim untuk bekerja dalam satu repository yang sama.

Setiap anggota dapat:
- Menambahkan file
- Mengedit kode
- Melakukan commit
- Berkontribusi dalam dokumentasi

---

## Penerapan pada Project
Dalam project sistem CBT ini, GitHub digunakan untuk:
- Menyimpan source code pada folder src
- Menyimpan dokumen SKPL pada folder SKPL
- Menyimpan hasil review pada folder review
- Menyimpan hasil pengujian pada folder testing

---

## Kesimpulan
Penggunaan Github sangat membantu dalam meningkatkan kualitas software, karena seluruh proses mulai dari pengembangan, review, hingga pengujian terdokumentasi secara rapi, terstruktur, dan mudah diakses.
